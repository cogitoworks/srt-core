# SRT Audit Note Template v0.1

## 概要

本テンプレートは、SRT（Semantic Reattribution Theory）を用いた価値帰属監査の出力形式を標準化するためのものである。SRT Core 0.7の手続きに基づき、今回のJ-POP論争分析（Case 001）での実践から抽出した。

---

## テンプレート構造

### 0. メタ情報

```
対象：[監査対象の名称・リンク]
監査者：[誰が行ったか]
日付：[YYYY-MM-DD]
入力範囲：[何を読んだか／何を読んでいないか]
動機：[なぜこの対象を選んだか]
```

**設計意図：** 入力範囲の限定は、分析の射程を明示し、射程外への批判を防御するために必須。動機の透明化は、SRT§12（自己監査可能性）の要請に応える。

---

### 1. 全体像

- この対象について、何が起きているように見えるかを1段落で記述する
- 勝敗判定や価値判断ではなく、「どんな種類の食い違い／混線が起きているか」を述べる

---

### 2. 主張の分解

対象から主張を抽出し、以下の形式で列挙する。

```
主張X：[主張の要約（1〜2文）]
出典：[どの記事／発話／テキストのどの部分から抽出したか]
```

**注記事項（必須）：**
- この分解は唯一の切り方ではなく、別の分解も可能であることを明記する
- 記事間の関係性（一次発信か応答か等）がある場合はここで示す

**分類カテゴリ（【3】で使用）：**

| カテゴリ | 定義 |
|---|---|
| 観測 | 参照可能なデータや事例に基づく記述。他者が確認・反論可能な水準にある主張 |
| 解釈 | 観測されたものに対する意味づけや構造的な読み。同じ観測から別の解釈も成立しうる主張 |
| 定義依存 | 結論がある概念の定義の取り方に左右される主張。定義が変われば判断が変わる |
| 総括 | 個別の観測や解釈を束ねて大きな結論を導く主張。飛躍の大きさが問われる |

---

### 3. SRTによる読み解き

各主張に対して以下の4項目を出す。

```
主張X：[主張の見出し]

分類：[観測 / 解釈 / 定義依存 / 総括 / またはその混合]

SRT的に受理しやすい部分：
[この主張のうち、参照可能な足場を持っている部分。
 どの発生層（作品内在層・誘発受容層・共同体増殖層・プラットフォーム増幅層）に
 アンカーがあるかを示す]

SRT的に保留すべき部分：
[この主張のうち、追加検証なしには確定できない部分。
 なぜ保留なのか（代替説明が成立する、定義依存である、再接続が不十分、等）を示す]

理由：
[上記の判断の根拠。可能であればSRTの該当セクション（§番号）を参照]
```

**運用上の注意：**
- 「受理」は「正しい」を意味しない。SRTの手続き上、参照可能な足場があるという意味
- 「保留」は「間違い」を意味しない。追加検証が必要、または定義選択に依存するという意味
- 全ての主張に必ず受理部分と保留部分の両方を書く。片方だけの主張があれば、なぜ片方しかないのかを理由欄で説明する

---

### 4. ズレの本体

- 主張間の食い違いや論争の構造を、発生層モデルと会計区分の観点から記述する
- 「内在 vs 外在」のような二項対立に回収されがちな場合、その回収で何が見えなくなるかを示す
- 一次対象アンカーの範囲定義が争点になっている場合は、それを明示する

---

### 5. 暫定結論

- どちらか一方を勝たせずに、この分析から何が見えるかをまとめる
- SRTとして断定できること、断定できないこと、今後の検証が必要なことを区別する

---

### 6. Open Checks

- 判断を保留すべき点を列挙する
- 未検証の入力（動画、コメント、関連記事等）があればここで明示する
- 主張の分解の恣意性についての注記もここに含める

---

## 使用上の制約

1. **このテンプレートはスコアリングシートではない。** Core/Border/Outerは数値ではなく、監査上の位置づけである
2. **一次対象アンカーの定義を省略して会計区分だけ出してはならない。** 対象ごとに何が一次的支持基盤かを先に定義する（SRT§6-1）
3. **テンプレートを埋めただけでSRT適用が完了したとは限らない。** テンプレートは手続きの骨格であり、各項目の判断には文脈依存的な検討が残る
4. **SRT§10のNOT境界を参照すること。** このテンプレートは格付け装置ではなく、好悪の理論語化でもなく、印象批評の代替でもない
