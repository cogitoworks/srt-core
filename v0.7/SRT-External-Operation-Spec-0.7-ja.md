# SRT External Operation Spec 0.7
## Semantic Reattribution Theory
### 外部適用崩壊リスク対策・適用成立条件・失格条件のための運用規約

**著者:** cogitoworks  
**バージョン:** 0.7  
**日付:** 2026-03  
**ライセンス:** CC BY-SA 4.0

---

## 0. 目的

本規約は、外部 LLM または第三者が SRT を適用する際に、SRT 語彙のみを用いた印象批評、格付け批評、好悪の理論語化へ崩壊することを防ぐための運用条件を定める。

ここでいう「外部適用崩壊」とは、Core / Border / Outer、過剰再帰属、共同体増殖層、プラットフォーム増幅層などの語彙が使用されていても、実際には SRT の監査手続きが起動していない、あるいは途中で別種の出力へ転化している状態を指す。

本規約の関心は、SRT を平易化することにはない。SRT 語彙を用いた雑な批評を、SRT 適用として成立させないための境界を定めることにある。

---

## 1. 問題定義

SRT は、価値の有無を断定する理論ではなく、価値の発生地点と帰属先の混線を監査する理論である。

しかし外部運用においては、しばしば次のような崩壊が生じる。

・監査対象となる価値主張が固定されない

・一次対象アンカー確認が省略される

・発生層の強さと会計区分が短絡される

・可搬性や代替可能性が印象語として乱用される

・NOT 境界が補助的注意書きとして処理される

・既存の審美判断やジャンル序列が SRT 語彙で再包装される

この状態は、SRT 適用ではない。SRT 語彙を用いた別種の批評である。

したがって、本規約は、外部出力が SRT によって何を判定したか以前に、そもそも SRT が起動していたか、途中で別物化していないかを先に問う。

---

## 2. 固定点

本規約は、以下を固定点とする。

第一に、SRT は作品全体または対象全体の一括格付け理論ではない。SRT の監査単位は、対象そのものではなく、その対象について提出された個別の価値主張である。

第二に、SRT は価値の存否判定理論ではない。SRT が問うのは、価値があるかないかではなく、その価値をどこへ計上するかである。

第三に、発生層と会計区分は同一ではない。共同体増殖層またはプラットフォーム増幅層が強いことは、それだけで Outer 確定を意味しない。

第四に、NOT は補助説明ではない。NOT は理論の雰囲気説明ではなく、越境時に別物化を判定する運用境界である。

第五に、SRT 語彙の使用は、SRT 適用成立の条件ではない。Core / Border / Outer 等の語が出ていても、手続きが欠けていれば不成立とみなす。

---

## 3. 状態区分

外部出力は、少なくとも次の三状態に区別される。

### 3.1 SRT未起動

SRT 語彙は出現していても、監査主張固定、一次対象アンカー要求、再接続確認、代替説明比較などの手続きが作動していない状態である。

ここでは語彙が借用されているのみであり、監査手続きそのものは開始されていない。

### 3.2 SRT誤起動

手続きが部分的には開始されているが、途中で会計区分の短絡、NOT 未検査、外部要因の列挙による即断定などが生じ、監査理論としての同一性を失っている状態である。

ここでは起動はしているが、運用の途中で別物化が進行している。

### 3.3 SRT適用失敗

手続きそのものは正しく起動しているが、資料不足、アンカー不足、代替説明未解決、争点過多などにより、Core / Border / Outer の暫定判定が安定しない状態である。

これは未起動や誤起動とは異なる。適用失敗は、手続きが成立していることを前提とした保留または判定不能である。

---

## 4. 適用前ゲート

外部 LLM または第三者が SRT を適用する際は、以下の前ゲートを必須とする。ひとつでも未充足なら、その出力は SRT 適用成立とみなさない。

### Gate-1 監査主張固定

まず監査対象となる価値主張を一文で固定すること。

例:

「この作品は現代社会への鋭い批判である」

「この作品の感動の大部分は本文に支えられている」

「この作品は文学的傑作である」

作品全体や対象全体を丸ごと判定対象にしてはならない。

### Gate-2 一次対象アンカー要求

固定した主張に対して、監査対象そのものの内部要素、すなわち一次的支持基盤へ戻るための具体要素を列挙すること。

小説であれば本文、映像であれば画面・構成・台詞・演出、ゲームであれば実装・UI・進行・システム、理論文であれば定義・論証接続・除外規定などがここに含まれる。

一次対象へ戻れない主張は、その時点で本体算入保留とする。

### Gate-3 再接続確認

列挙した要素が、本当にその主張へ接続しているかを確認すること。

「その要素が存在する」ことと、「その価値が対象本体の仕事である」ことは同一ではない。要素の存在確認だけで本体算入へ進んではならない。

### Gate-4 代替説明比較

同じ価値主張が、対象本体以外の説明でも同程度以上に成立しないかを比較すること。

少なくとも次の候補は比較対象に含める。

・共同体解釈増殖

・読前期待／事前文脈

・制度的権威付与

・プラットフォーム増幅

・ジャンル慣習による補完

・読者側補完の一般性

### Gate-5 会計区分判定

上記の手続きを経た後にのみ、Core / Border / Outer の暫定判定を与えてよい。

順序を逆転してはならない。先に会計区分を置いてから理由を収集する運用は、SRT の手続きに反する。

### Gate-6 Open Checks 明示

未確認事項、保留点、判定不能点、争点残留を必ず残すこと。

閉じた断定で終えてはならない。

---

## 5. 失格条件

以下のいずれかに当てはまる出力は、SRT 適用として失格とする。

① 監査主張を固定せず、作品全体または対象全体を一括で Core / Border / Outer に置くこと。

② 一次対象アンカーなしに、「共同体増殖層が強いから Outer」「話題性が高いから Outer」などと短絡すること。

③ BookTok、受賞歴、SNS 話題性、肩書き、バズワードなどを挙げただけで、過剰再帰属を断定すること。

④ 可搬性や代替可能性を、単なる印象的言い換え可能性として処理すること。

⑤ ジャンル序列、個人的好悪、既存評価、文化的ポジション取りを、そのまま SRT 判定として再包装すること。

⑥ 「深い」「浅い」「普通」「薄い」などの審美語を、一次対象アンカー確認や代替説明比較を飛ばして会計区分へ直結させること。

⑦ NOT 違反検査を行わずに、応用例や判定例のみを量産すること。

⑧ Open Checks を出さず、即断定で締めること。

これらは、結論内容の好悪による失格ではない。監査理論としての起動条件が満たされていない、または途中で崩壊しているという意味での失格である。

---

## 6. NOT 境界の運用再固定

本規約において、以下を SRT 外部運用上の NOT として再固定する。

SRT は、作品ランキング装置ではない。

SRT は、ジャンル序列正当化装置ではない。

SRT は、バズ作品切り下げ理論ではない。

SRT は、高尚／低俗の再命名ではない。

SRT は、印象批評の理論語化ではない。

SRT は、BookTok や SNS 流行を見ただけで Outer 認定する近道ではない。

SRT は、一次対象読解を省略して会計区分だけを提示するショートカットではない。

SRT は、好みを監査結果として偽装するための語彙集合ではない。

ここでの NOT は、否定修辞ではなく、理論同一性を保全するための運用境界である。

---

## 7. 外部 LLM 向け出力制約

外部 LLM に SRT を適用させる場合、自由記述先行を禁止する。

理由は、自然言語上の流暢さだけでは、手続き未起動の出力であっても、監査が成立しているように見えうるからである。

したがって、外部適用時には、先に固定フォーマットを充足させ、その後にのみ総評や評論的記述を許可する。

---

## 8. 固定監査フォーマット

外部 LLM または第三者は、SRT を適用する際、最低限次の形式を先に埋めなければならない。

**【SRT 暫定監査フォーマット】**

1. 監査対象主張
2. 一次対象アンカー候補
3. 主張への再接続可否
4. 代替説明候補
5. 発生層メモ
6. Core / Border / Outer 暫定判定
7. NOT 違反チェック
8. Open Checks

自由な総評は、このフォーマット充足後にのみ許可される。

目的は、自然言語上の流暢さより先に、監査手続きの起動を強制することにある。

---

## 9. 本文との関係

本規約は、SRT Core 0.7 本文の代替ではない。

本文が担うのは、価値帰属監査理論としての定義、発生層、会計区分、四軸、認識論的位置づけ、NOT、自己監査可能性である。

本規約が担うのは、外部運用時にそれらが別物化せずに起動するための成立条件、失格条件、運用制約である。

したがって、SRT を外部へ渡す場合、本文のみを渡してはならない。少なくとも、本文と本規約は対で提示される必要がある。

---

## 10. Open Checks

現時点で残っている主要な検討課題は以下である。

第一に、前ゲートを増やしすぎると、運用負荷が高まり実用性が落ちる可能性がある。どこまでを必須起動条件とし、どこからを精密監査条件とするかは未固定である。

第二に、一次対象アンカーの粒度をどこまで要求するかは未固定である。媒体別・対象別の具体性基準が必要である。

第三に、可搬性と代替可能性の定義精度はさらに必要である。外部運用では両者がしばしば混同される。

第四に、外部 LLM に対して NOT をどの形式で与えると最も境界として機能するかは未検証である。

第五に、SRT 未起動、SRT 誤起動、SRT 適用失敗の区別を、どこまで安定運用できるかは、実例蓄積を要する。

第六に、固定フォーマットが硬直化しすぎると、監査支援ではなく記入儀式へ堕する危険がある。運用性と境界保全の均衡が必要である。

---

## 11. 結語

SRT の問題は、理論語彙が不足していることではない。むしろ逆である。語彙だけでも監査成立に見える出力を生成できてしまうことが、外部崩壊の一因になる。

したがって、SRT を外部へ出す際に必要なのは、より流暢な説明より先に、何が適用成立であり、何が未起動であり、何が誤起動であり、何が失格であるかを明文化することである。

本規約は、そのための外周境界である。
